1. Comunicação clara e constante
    Prática: Definir canais oficiais (Slack, Teams, e-mail) e manter atualizações regulares.
    Exemplo: “Postar no canal do projeto um resumo diário das tarefas concluídas e pendentes.”
2. Gestão de tarefas e responsabilidades
    Prática: Usar ferramentas de organização (Trello, Jira, Notion) para atribuir tarefas.
    Exemplo: “Cada membro deve atualizar o status da sua tarefa ao final do dia.”
3. Uso de Git e versionamento
    Prática: Criar branches específicas para cada funcionalidade ou correção.
    Exemplo: feature-login ou bugfix-header.
    Prática: Fazer commits pequenos e descritivos.
    Exemplo: git commit -m "Adiciona validação de formulário de login"
4. Respeito ao tempo e às ideias dos outros
    Prática: Agendar reuniões curtas e objetivas, respeitando horários.
    Exemplo: “Reunião de 15 minutos às segundas para alinhar prioridades.”
    Prática: Escutar antes de propor soluções.
    Exemplo: “Deixar cada membro expor sua ideia antes de discutir alternativas.”
5. Documentação mínima e útil
    Prática: Manter README atualizado com instruções de instalação e uso.
    Exemplo: “README inclui passos para rodar o projeto localmente e links para recursos externos.”
    Prática: Registrar decisões importantes.
    Exemplo: “Adicionar no Notion a justificativa para escolha da paleta de cores.”
6. Feedback construtivo
    Prática: Revisar código ou conteúdo com foco em melhorias, não em críticas pessoais.
    Exemplo: “Sugiro usar CSS Grid aqui para simplificar o layout, o que acha?”
7. Valorização da diversidade
    Prática: Incentivar diferentes perspectivas e backgrounds.
    Exemplo: “Abrir espaço para que cada colaborador explique como sua experiência pode enriquecer o projeto.”
